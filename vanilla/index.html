Gems of Quote

  • an "app" is actually a collection of features
  • start with a piece of actual functionality
  • hell, you might not even need that other stuff anyways
  • resist the temptation to introduce color right away
  • leave sketches and wireframes behind when you've made a decision
  • you don't need to design every single feature in an app before you move on to implementation
  • work in short cycles
  • Build the real thing as early as possible so your imagination doesn't have to do all the heavy lifting
  • don't imply functionality in your designs that you aren't ready to build
  • designing the smallest useful version you can ship reduces that risk considerably
  • if part of a feature is a "nice-to-have", design it later
  • the words you use in an interface have a massive influence on the overall personality
  • you don't want to look like a second-rate version of something else
  • choose from a set of 8-10 shades picked out ahead of time
  • define a restrictive type scale in advance
  • the more systems you have in place, the faster you'll be able to work
  • visual hierarchy is the most effective tool you have for making something feel "designed"
  • deliberately de-emphasize secondary and tertiary information and hightlight the elements that are important
  • primary content that's too large, and secondary content that's too small
  • use a softer color for supporting text instead of a tiny font size
  • trap of displaying it using a naive label: value format
  • avoid adding a label by adding clarifying text to the value
  • a lot of time, section titles act more like labels than headings
  • pick elements for semantic purposes and style them however you need to create the best visual hierarchy
  • every action on a page sits somewhere in a pyramid of importance
